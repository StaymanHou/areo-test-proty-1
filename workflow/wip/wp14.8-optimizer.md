# Feature: WP14.8 — Score function + Nelder-Mead optimizer + CLI

## Problem Statement

D14 cascade step 3 of 3. WP14.7 shipped the Node Rapier-WASM harness as an exported pure function `runHarness({fixture, ticks, params, configPath?}): string` at `tools/tune/harness.ts:162-187`. WP14.8 builds the **search half**: a multi-regime score function over harness-emitted trajectories, a dimension-agnostic Nelder-Mead optimizer with K random restarts and local quadratic regression on the best simplex, and a CLI entry that ties them together with a results JSON output.

Score function is the load-bearing piece per `feedback_verify_self_envelope.md` — WP14.5's original failure was a 1-throttle observation hiding 3-throttle divergence. The score MUST be a multi-regime weighted sum with a NaN penalty that encodes time-to-first-NaN as a gradient (per D14.4) so the optimizer can move *toward* later-NaN regions when every sampled point explodes.

Downstream unblocks rescoped WP14.5 (β5 + β4 joint tuning per SURFACE-2026-05-16-01 + SURFACE-2026-05-12-03), which gates WP15/WP16. Per SURFACE-2026-05-16-01 the optimizer must handle ≥4 knobs from day 1.
