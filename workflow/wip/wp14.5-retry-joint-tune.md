# Feature: WP14.5-retry — Joint (clQ, clAlphaDot) tuning via npm run tune

## Problem Statement

The D14 cascade (WP14.6 + WP14.7 + WP14.8) is fully landed, producing a working `npm run tune` CLI with deterministic Nelder-Mead + restart + regression-gradient output. Two open `high`-priority SURFACE items — SURFACE-2026-05-12-03 (β5 `clAlphaDot` divergence) and SURFACE-2026-05-16-01 (β4 `clQ` explicit-Euler instability above V_REF) — both route their resolution through this WP. The mechanism-side hypothesis (Option D in SURFACE-2026-05-16-01, mechanism-tuning in -05-12-03) is that **a stable point in the joint (clQ, clAlphaDot) parameter space may exist for wing-left and h-stab** when searched systematically across regimes the prior hand-tuning failed to cover. Run the optimizer over that joint space; if it finds a point that satisfies the multi-regime envelope, commit `aircraft.json` and un-skip `tests/e2e/phugoid-probe.spec.ts`. If it does not, escalate with the optimizer's regression-Hessian + per-regime trajectories as evidence for which Option (A/B/C of -05-12-03; A/B/C of -05-16-01) is best.

**Acceptance threshold (recorded at plan time):**
- All 3 regimes (low/mid/high throttle = 0.05/0.15/0.4) produce **finite-trajectory** trajectories — no NaN through the 1800-tick (30s) probe window.
- **Total score ≥ -300** (sum of 3 regime scores; per-regime average ≥ -100). The DEFAULT_ENVELOPES per-regime score is `-(altPen² + asPen² + prPen² + phuPen)`; a regime fully within envelope scores 0, a regime modestly outside scores roughly -10 to -100, NaN scores -1e9. Threshold of -300 total means modest envelope excursions are tolerated as long as no regime catastrophically fails.

**Branch policy on Phase 1 outcome:**
- **Cross-threshold:** Phase 2A — commit aircraft.json + un-skip probe spec.
- **Below-threshold:** Phase 2B — file SURFACE escalation with optimizer artifacts (results JSON + per-regime CSV trajectories); leave aircraft.json untouched.

Critical observation: restart 2's h-stab clAlphaDot is -0.79 (near zero), and ALL FOUR restarts in the box `[0..20] × [-10..20] × [0..20] × [-10..20]` hit identical NaN floor. The decoded floor score `-1e9 + nanTick` averaged across 3 regimes is `-2,999,999,985 = -3e9 + 15`, meaning all 3 regimes NaN at average tick ~5 (essentially immediately).

### Regression Hessian — DEGENERATE
- `regression: null` in the results JSON. Per `optimizer.ts`, the quadratic-regression fit requires variance across the simplex's score samples to produce a stable gradient/Hessian. Here every simplex vertex scores at the NaN floor, so variance is zero (or trivially +15 from NaN-tick differences), and the regression solver returns null. This is itself diagnostic evidence: **no local quadratic shape exists in the joint (clQ, clAlphaDot) space at these scales — the entire 4D box is NaN-poisoned.**

### Why no point in the box meets threshold
- Any non-zero clAlphaDot at typical magnitudes (≥0.1) instantly destabilizes via the raw `dα/dt` mechanism per SURFACE-2026-05-12-03's startup-transient analysis. The "no-op in working regime" lesson from `feedback_asymmetric_fix_no_op.md` is violated by the mechanism itself.
- The baseline clQ=3,8 with clAD=0 holds low/mid stable but NaN's in high-throttle (β4 explicit-Euler instability above V_REF per SURFACE-2026-05-16-01).
- Lower clQ=1 ALSO NaN's across all regimes — because the ω×r damping baseline mechanism *needs* `(1 + clQ)` amplification to stabilize at all. clQ near zero collapses WP6.5's stability.
- Higher clQ (12-17 in the optimizer's restarts) NaN's faster — more β4 explicit-Euler instability.
- The window of (clQ, clAlphaDot) values that produces a finite high-throttle trajectory is empirically **empty** for the current mechanism formulation.

### Decision per `feedback_retune_attempt_budget.md`
4 optimizer restarts + 4 hand-picked points = 8 distinct evaluations across the 4D parameter space. All below threshold. Per the active-recall discipline: this matches the "refute in 2-3 attempts then escalate" rule — the optimizer IS the systematic version of that budget, and 4 random restarts produced ZERO informative gradient (all NaN floor). Continuing to search would burn budget without information. **Escalate to mechanism revision.**

### Which mechanism option does the evidence support?

Per SURFACE-2026-05-16-01 (β4 instability above V_REF):
- **Option A (implicit Euler for damping)** — strongly supported. The empirical pattern (NaN driven by β4 stiff-damper sign-flips above V_REF, identical to the SURFACE-2026-05-16-01 diagnostic) is *exactly* what implicit-Euler integration of stiff damping is designed to fix.
- **Option B (magnitude clamp)** — possible half-measure but the optimizer found NO clQ value that's stable in high throttle, suggesting the magnitude problem is fundamental, not a tail-clipping issue.
- **Option C (vScale clamp)** — same as B in spirit.
- **Option D (lower clQ via harness search)** — **REFUTED by this WP**. The optimizer's 4 restarts and the 4 hand-picked probes collectively rule out the "find a stable clQ at dt=1/60" hypothesis. Lower clQ destabilizes ω×r baseline; higher clQ destabilizes β4-amplified damper.

Per SURFACE-2026-05-12-03 (β5 divergence):
- **Option A (non-dim form `c̄/(2V)`)** — supported. The tiny clAD=0.1 probe shows immediate divergence (NaN at tick 85-199) even with WP6.5-stable clQ, exactly the "raw dα/dt is dimensionally wrong" hypothesis.
- **Option B (magnitude clamp on dα/dt)** — possible but the non-dim form is the standard aerodynamic treatment and addresses the same underlying issue.
- **Option C (sign flip)** — REFUTED in original WP14.5; restart 3's clAD_h=-9.88 still NaN's, consistent with that finding.

**Recommended escalation:** Option A on both β4 (implicit-Euler form) and β5 (non-dim form). These are independent mechanism-design issues — each is its own arch revision candidate.
