## Plan-time physics derivation (CLAUDE.md Rule #5)

Performed BEFORE re-reading arch.md D18 verification-section prose, per Rule #5's operator-as-architect-side honesty intent.

1. **Energy balance under D18 (textbook drag polar).** Terminal velocity in unpowered dive: `0.5·ρ·V_t²·(CD_0·S + CD_0,f·A_f + k·CL²·S) = m·g·sin(γ)`. Pre-D18: only `CD_0·S` term active → `V_t ≈ 245 m/s` (computed in SURFACE-23-01 evidence). With D18 at hand-probe values (`k=0.3, k=0.5, cd0=0.5, area=2.0`): expected `V_t ≈ √(2·m·g / (ρ·(CD_0·S + 0.5·2.0 + (0.3·S_w + 0.5·S_h)·CL²)))`. At CL=0.7 (typical level-flight) and S_w=6, S_h=1.5: `(0.3·6 + 0.5·1.5)·0.49 = 1.30`. Total drag area ≈ `0.02·13.5 + 1.0 + 1.30·0.49 ≈ 1.91 m²`. V_t ≈ `√(2·1000·9.81 / (1.225·1.91)) ≈ 91 m/s`. Matches the WP14.11.5 hand-probe observation (peak AS 29.99 m/s at throttle-low with thrust subtracting). ✓ Dimensional + numerical sanity OK.
2. **Joint-tune dimensionality risk.** 8-dim search × 4 restarts × Nelder-Mead simplex = ~9 simplex-vertices per restart × ~50 iterations average = ~1800 harness evaluations × 3 regimes × 1800 ticks each = ~9.7M physics ticks. Harness is ~80 ticks/sec (single-threaded JS) → wall time estimate ~30-60 min. Acceptable for one session.
3. **Verify-self regime sensitivity.** WP14.10 (D16 mechanism WP) substituted throttle-low + throttle-mid (skipped throttle-high) because baseline NaN'd at throttle-high; CLAUDE.md Rule #4 plan-time addendum (2026-05-23) explicitly licenses this. **For WP14.12 the regimes are DEPLOYMENT regimes (low + mid + high — all 3), not mechanism-isolation regimes.** The 1800-tick gate is the deployment criterion; if any of the 3 regimes NaN at the optimizer's best, criterion 1 fails and we're on Branch B. The harness's `--regimes low,mid,high` already covers this.

**Derivation matched arch.md D18 verification-section prose.** No errata detected. Informational plan-time prior — not a SURFACE.
